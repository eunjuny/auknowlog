# 쿠버네티스 기본 지식 퀴즈
총 10문항 · 정답 10 · 오답 0

## Q1. 쿠버네티스(Kubernetes)의 주된 목적은 무엇인가요?
- A. 데이터베이스 관리
- B. 분산 시스템의 컨테이너화된 워크로드 자동화, 배포 및 확장 관리
- C. 네트워크 보안 강화
- D. 웹 프레임워크 개발
✅ 내 답: 분산 시스템의 컨테이너화된 워크로드 자동화, 배포 및 확장 관리
설명: 쿠버네티스는 컨테이너화된 애플리케이션을 자동으로 배포, 확장 및 관리하는 오픈소스 시스템입니다.

## Q2. 쿠버네티스 클러스터의 '컨트롤 플레인'에 속하는 핵심 구성 요소가 아닌 것은 무엇인가요?
- A. kube-apiserver
- B. etcd
- C. kube-scheduler
- D. kubelet
✅ 내 답: kubelet
설명: kubelet은 워커 노드에서 실행되며 컨테이너 런타임과 통신하여 파드를 관리하는 에이전트입니다. kube-apiserver, etcd, kube-scheduler는 컨트롤 플레인의 핵심 구성 요소입니다.

## Q3. 쿠버네티스에서 컨테이너화된 애플리케이션의 가장 작은 배포 단위는 무엇인가요?
- A. Node
- B. Cluster
- C. Pod
- D. Deployment
✅ 내 답: Pod
설명: 파드(Pod)는 하나 이상의 컨테이너 그룹(공유 스토리지, 네트워크 자원, 컨테이너 실행 방법을 명시하는 옵션 포함)을 나타내는 쿠버네티스에서 생성되고 관리될 수 있는 가장 작은 배포 단위입니다.

## Q4. 쿠버네티스에서 여러 개의 동일한 파드 인스턴스를 유지하고 파드 장애 시 자동으로 복구하는 역할을 담당하는 오브젝트는 무엇인가요?
- A. Service
- B. Ingress
- C. Deployment
- D. ConfigMap
✅ 내 답: Deployment
설명: 디플로이먼트(Deployment)는 파드와 레플리카셋(ReplicaSet)을 관리하며, 파드의 선언적 업데이트와 확장을 가능하게 합니다.

## Q5. 쿠버네티스 클러스터 외부 또는 내부에서 파드 그룹에 접근할 수 있도록 안정적인 네트워크 엔드포인트를 제공하는 오브젝트는 무엇인가요?
- A. Volume
- B. Namespace
- C. Service
- D. Secret
✅ 내 답: Service
설명: 서비스(Service)는 파드 그룹에 대한 네트워크 접근을 추상화하여, 파드의 IP 주소가 변경되어도 지속적으로 접근 가능한 엔드포인트를 제공합니다.

## Q6. 쿠버네티스에서 리소스를 논리적으로 분리하고 격리하여 관리하는 데 사용되는 메커니즘은 무엇인가요?
- A. Label
- B. Annotation
- C. Namespace
- D. Taint
✅ 내 답: Namespace
설명: 네임스페이스(Namespace)는 클러스터 내의 리소스를 논리적으로 분리하여 여러 팀이나 프로젝트가 하나의 클러스터를 공유할 때 리소스 충돌을 방지하고 관리 효율성을 높입니다.

## Q7. 쿠버네티스에서 'etcd'의 주요 역할은 무엇인가요?
- A. 컨테이너 이미지 저장
- B. 클러스터 상태 및 설정 데이터 저장 (분산 키-값 스토어)
- C. 컨테이너 런타임 환경 제공
- D. 네트워크 트래픽 로드 밸런싱
✅ 내 답: 클러스터 상태 및 설정 데이터 저장 (분산 키-값 스토어)
설명: etcd는 쿠버네티스 클러스터의 모든 상태 데이터(예: 스케줄링 정보, 서비스, 파드, 설정 데이터)를 저장하는 분산 키-값 스토어입니다.

## Q8. 'kubectl' 명령어의 주된 용도는 무엇인가요?
- A. 쿠버네티스 소스 코드 컴파일
- B. 클러스터 노드 하드웨어 모니터링
- C. 쿠버네티스 클러스터와 통신하여 리소스 관리
- D. 컨테이너 이미지 빌드
✅ 내 답: 쿠버네티스 클러스터와 통신하여 리소스 관리
설명: kubectl은 쿠버네티스 클러스터의 컨트롤 플레인에 명령을 보내 클러스터 리소스(파드, 서비스, 디플로이먼트 등)를 배포, 관리, 검사하는 데 사용되는 커맨드 라인 도구입니다.

## Q9. 쿠버네티스 워커 노드에서 실행되며, 파드의 생명 주기를 관리하고 컨테이너 런타임과 통신하는 에이전트는 무엇인가요?
- A. kube-apiserver
- B. kubelet
- C. kube-proxy
- D. kube-scheduler
✅ 내 답: kubelet
설명: kubelet은 각 워커 노드에서 실행되며, 컨트롤 플레인으로부터 파드 스펙을 받아 해당 노드의 컨테이너 런타임(Docker, containerd 등)을 통해 파드를 생성, 시작, 중지 및 관리합니다.

## Q10. 쿠버네티스에서 파드의 컨테이너 간에 공유되는 스토리지를 제공하는 오브젝트는 무엇인가요?
- A. ConfigMap
- B. Secret
- C. Volume
- D. PersistentVolumeClaim
✅ 내 답: Volume
설명: 볼륨(Volume)은 파드 내의 컨테이너들이 데이터를 공유하고, 파드가 종료되거나 재시작될 때도 데이터가 유지될 수 있도록 스토리지를 제공합니다. PersistentVolumeClaim은 동적/정적 프로비저닝을 통해 클러스터 리소스로서의 볼륨을 요청합니다.
