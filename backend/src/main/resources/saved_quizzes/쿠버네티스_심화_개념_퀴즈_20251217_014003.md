# 쿠버네티스 심화 개념 퀴즈
총 4문항 · 정답 4 · 오답 0

## Q1. 쿠버네티스 클러스터에서 사용자나 서비스 계정에게 특정 리소스에 대한 접근 권한을 세밀하게 제어할 수 있도록 하는 보안 메커니즘은 무엇인가요?
- A. NetworkPolicy
- B. SecurityContext
- C. Role-Based Access Control (RBAC)
- D. PodSecurityPolicy
✅ 내 답: Role-Based Access Control (RBAC)
설명: RBAC(Role-Based Access Control)는 사용자나 서비스 계정에 역할을 할당하고, 그 역할에 따라 리소스에 대한 생성, 읽기, 업데이트, 삭제 등의 접근 권한을 정의하여 세밀한 권한 제어를 가능하게 합니다.

## Q2. 쿠버네티스에서 'PersistentVolume (PV)'과 'PersistentVolumeClaim (PVC)'의 관계는 어떻게 되나요?
- A. PV는 네트워크 설정이고, PVC는 방화벽 규칙이다.
- B. PV는 클러스터 관리자가 프로비저닝한 물리적 스토리지이고, PVC는 사용자가 그 스토리지를 요청하는 방식이다.
- C. PV는 컨테이너 이미지이고, PVC는 이미지 빌드 명령이다.
- D. PV는 노드의 메모리이고, PVC는 CPU 할당이다.
✅ 내 답: PV는 클러스터 관리자가 프로비저닝한 물리적 스토리지이고, PVC는 사용자가 그 스토리지를 요청하는 방식이다.
설명: PV는 클러스터 내의 추상화된 스토리지 자원을 나타내고, PVC는 사용자가 필요한 스토리지 리소스(크기, 접근 방식 등)를 요청하는 용도입니다.

## Q3. 쿠버네티스 '레플리카셋(ReplicaSet)'의 핵심 기능은 무엇인가요?
- A. 컨테이너의 네트워크 트래픽을 모니터링
- B. 지정된 수의 파드 복제본이 항상 실행되도록 보장
- C. 클러스터 노드의 상태를 감시
- D. 새로운 컨테이너 이미지를 빌드
✅ 내 답: 지정된 수의 파드 복제본이 항상 실행되도록 보장
설명: 레플리카셋은 항상 지정된 수의 파드 복제본이 실행되도록 보장하여, 애플리케이션의 가용성을 유지합니다.

## Q4. 쿠버네티스 클러스터의 '원하는 상태'를 모니터링하고 유지하는 책임이 있는 핵심 컴포넌트는 무엇입니까?
- A. Kubelet
- B. Kube-proxy
- C. API 서버 (API Server)
- D. 컨트롤러 매니저 (Controller Manager)
✅ 내 답: 컨트롤러 매니저 (Controller Manager)
설명: 컨트롤러 매니저는 클러스터의 현재 상태를 지속적으로 모니터링하고, 사용자가 정의한 '원하는 상태'와 일치하도록 여러 컨트롤러(예: ReplicaSet Controller)를 실행합니다.
