# 쿠버네티스 핵심 개념 퀴즈
총 10문항 · 정답 10 · 오답 0

## Q1. 쿠버네티스(Kubernetes)의 주요 역할은 무엇인가요?
- A. 분산 트랜잭션 관리
- B. 콘테이너화된 워크로드와 서비스를 관리하기 위한 이식 가능하고 확장 가능한 오픈 소스 플랫폼
- C. 운영체제 커널 개발
- D. 네트워크 방화벽 설정
✅ 내 답: 콘테이너화된 워크로드와 서비스를 관리하기 위한 이식 가능하고 확장 가능한 오픈 소스 플랫폼
설명: 쿠버네티스는 컨테이너화된 애플리케이션의 배포, 확장 및 관리를 자동화하는 오픈 소스 컨테이너 오케스트레이션 플랫폼입니다.

## Q2. 쿠버네티스 클러스터에서 가장 작은 배포 단위는 무엇인가요?
- A. Node
- B. Service
- C. Pod
- D. Deployment
✅ 내 답: Pod
설명: Pod는 쿠버네티스에서 생성되고 관리될 수 있는 배포 가능한 가장 작은 컴퓨팅 단위입니다. 하나 이상의 컨테이너(예: Docker 컨테이너)를 포함할 수 있습니다.

## Q3. 쿠버네티스 마스터 노드의 핵심 컴포넌트 중, 클러스터의 모든 데이터를 저장하는 분산 키-값 저장소는 무엇인가요?
- A. kube-apiserver
- B. kube-scheduler
- C. etcd
- D. kube-controller-manager
✅ 내 답: etcd
설명: etcd는 쿠버네티스 클러스터의 모든 설정 데이터, 상태, 메타데이터 등을 저장하는 분산 키-값 저장소입니다. 클러스터의 '뇌'와 같은 역할을 합니다.

## Q4. 애플리케이션의 집합(예: Nginx 웹 서버)을 쿠버네티스 클러스터에 배포하고 관리하기 위한 오브젝트는 무엇인가요?
- A. ConfigMap
- B. Secret
- C. Deployment
- D. Service
✅ 내 답: Deployment
설명: Deployment는 Pods와 ReplicaSets의 선언적 업데이트를 제공합니다. 이를 통해 애플리케이션의 배포, 롤백, 확장 등을 쉽게 관리할 수 있습니다.

## Q5. 쿠버네티스에서 Pods 집합에 대한 네트워크 접근을 추상화하여, Pods가 재생성되거나 IP 주소가 변경되어도 지속적인 접근이 가능하게 하는 오브젝트는 무엇인가요?
- A. Ingress
- B. Service
- C. Namespace
- D. Volume
✅ 내 답: Service
설명: Service는 Pods의 논리적 집합과 해당 Pods에 접근하는 정책을 정의합니다. 외부 트래픽을 Pods로 라우팅하거나 클러스터 내부에서 다른 애플리케이션이 접근할 수 있도록 안정적인 엔드포인트를 제공합니다.

## Q6. 쿠버네티스 클러스터에서 워커 노드(Worker Node)의 주요 컴포넌트 중, 각 노드에서 컨테이너 런타임과 통신하여 Pods를 실행하고 관리하는 에이전트는 무엇인가요?
- A. kube-proxy
- B. kubelet
- C. kube-apiserver
- D. etcd
✅ 내 답: kubelet
설명: kubelet은 클러스터의 각 노드에서 실행되는 에이전트로, API 서버로부터 PodSpec을 받아 Pods 내의 컨테이너가 올바르게 실행되고 있는지 확인하고 관리합니다.

## Q7. 쿠버네티스에서 외부 트래픽을 클러스터 내의 서비스로 라우팅할 때 사용하며, 보통 HTTP/HTTPS 기반의 L7 라우팅을 담당하는 오브젝트는 무엇인가요?
- A. Service
- B. Route
- C. Ingress
- D. NodePort
✅ 내 답: Ingress
설명: Ingress는 외부에서 클러스터 내의 서비스로 HTTP 및 HTTPS 라우팅을 제공합니다. 이는 단일 IP 주소로 여러 서비스를 노출할 수 있게 하며, L7 로드 밸런싱 기능을 제공합니다.

## Q8. 쿠버네티스에서 여러 팀이나 프로젝트가 하나의 클러스터를 공유할 때, 리소스(예: CPU, 메모리)를 격리하고 관리할 수 있도록 논리적으로 분할하는 단위는 무엇인가요?
- A. Cluster
- B. Node
- C. Namespace
- D. Region
✅ 내 답: Namespace
설명: Namespace는 쿠버네티스 클러스터 내에서 리소스 범위를 제공합니다. 이를 통해 여러 사용자가 하나의 클러스터를 공유할 때 리소스를 격리하고 관리할 수 있습니다.

## Q9. 쿠버네티스에서 Pod가 삭제되거나 재시작되어도 데이터가 손실되지 않도록 영구 스토리지를 제공하는 방식은 무엇인가요?
- A. Ephemeral Volume
- B. ConfigMap
- C. PersistentVolume / PersistentVolumeClaim
- D. Secret
✅ 내 답: PersistentVolume / PersistentVolumeClaim
설명: PersistentVolume (PV)은 클러스터 관리자가 프로비저닝한 스토리지의 한 부분이며, PersistentVolumeClaim (PVC)은 사용자가 PV를 요청하는 것입니다. 이 둘을 통해 Pods의 생명주기와 독립적인 영구 스토리지를 사용할 수 있습니다.

## Q10. 쿠버네티스 클러스터의 Master Node(Control Plane)에 속하지 않는 컴포넌트는 무엇인가요?
- A. kube-apiserver
- B. kube-scheduler
- C. kubelet
- D. kube-controller-manager
✅ 내 답: kubelet
설명: kubelet은 워커 노드(Worker Node)에서 실행되는 에이전트이며, kube-apiserver, kube-scheduler, kube-controller-manager는 마스터 노드의 핵심 컴포넌트들입니다.
